# 局域网文件传输

一个零配置的局域网文件传输工具。启动后，同一局域网内的任意设备（电脑、手机、平板）用浏览器打开即可上传、下载、删除文件，无需安装客户端。

## 特性

- **零客户端**：任何带浏览器的设备都能用，扫码即连
- **拖拽 / 粘贴上传**：支持多文件拖拽或 Ctrl+V 粘贴，带实时进度条
- **共享投放区**：任一设备上传的文件，其他设备都能看到并下载
- **匿名聊天室**：随机昵称、气泡式 UI、SSE 实时推送，局域网内即时通讯，消息持久化重启不丢失
- **实时在线人数**：聊天室显示当前在线设备数（基于 SSE 连接），断线自动标记
- **开机自启**：Web 页面一键开启，重启后自动运行（Windows 通过 Startup 实现，无控制台窗口）
- **自定义保存目录**：支持 `--dir` 参数、环境变量、Web UI 三种方式配置
- **速率限制**：按 IP 滑动窗口限流，防滥用
- **操作日志**：上传/下载/删除操作自动记录到日志文件
- **路径安全**：内置路径穿越防护，无法读取共享目录之外的文件

## 安装

需要 Python 3.9+。

```bash
pip install -r requirements.txt
```

Windows 用户也可直接下载 [Releases](https://github.com/Yi-D1Bug/lan-drop/releases) 中的 `lan-drop.exe`，双击运行，无需安装 Python。

## 使用

```bash
python app.py                     # 默认端口 9000，保存到 ./shared
python app.py --port 8200         # 自定义端口
python app.py --dir D:\my-files   # 自定义保存目录
python app.py --host 127.0.0.1 --port 8200   # 仅本机访问
```

优先级：CLI 参数 > 环境变量 > Web UI 配置 > 默认值。

启动后终端会打印局域网访问地址和二维码：

```
================================================
  局域网文件传输 v1.4.1 已启动
  共享目录: F:\claude\Lan\shared
  本机访问: http://127.0.0.1:9000
  局域网访问: http://192.168.1.5:9000
================================================
```

手机扫码或在浏览器输入「局域网访问」地址即可。上传的文件保存在共享目录中。

### 自定义保存目录

三种方式（按优先级排序）：

```bash
# 方式 1: CLI 参数
python app.py --dir /path/to/files

# 方式 2: 环境变量
LAN_SHARE_DIR=/path/to/files python app.py

# 方式 3: Web UI 设置面板（页面底部 ⚙️ → 保存目录 → 重启生效）
```

### 开机自启

Web 页面底部 ⚙️ 设置面板 → 开启「开机自启」。下次系统启动时自动运行，无控制台窗口。

关闭自启：设置面板中关掉开关即可。

### 配置文件

所有设置（自启、目录、端口等）持久化在：

- **Windows**: `%APPDATA%\lan-drop\config.json`
- **Linux/Mac**: `~/.config/lan-drop/config.json`

### 聊天记录与操作日志

- 聊天消息保存在共享目录下的 `.chat.json`，服务重启不丢失
- 操作日志保存在共享目录下的 `.lan-drop.log`（TSV 格式：时间、IP、动作、文件名）

这些文件以 `.` 开头，不会显示在 Web 文件列表中。

## ⚠️ 安全须知

本工具**不带任何鉴权**：凡是能访问到该地址的设备，都可以上传、下载和删除文件。这对于可信局域网内的临时投放区是符合预期的设计。

- ✅ 适用：家庭、办公室等可信局域网
- ❌ **切勿**将本工具暴露到公网，或在路由器上做端口映射 —— 这等于向所有人开放一个可任意读写文件的入口

如需在不完全可信的网络中使用，请自行在前置反向代理（Nginx 等）上配置访问控制。
